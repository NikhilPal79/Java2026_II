# API [application programming interface ][with api we can connect one application to another application ] 
# Type of api 
1. SOAP API [SIMPLE OBJECT ACCESS PROTOCOL API]
2. REST API [REPRESENTATIONAL STATE TRANSFER][Simple API form. Provide interoperability between applications over network. Mainly used in web and mobile apps]
3. RPC API [REMOTE PROCEDURE CALL ]

# They all use HTTP METHOD to communicate but use JSON format to SEND and RECEIVE the response.

# CONSTRAINTS OF THE REST API 
1. statelessness [client/server are unaware of the state of each other]




# HTTP METHOD 
1. GET 
2. POST 
3. PUT 
4. PATCH
5. DELETE 
6. QUERY [NEW IN 2026] 

# URL [Uniform Resource Locator ]
1. BASE URL
2. END POINT 
3. QUERY PARAMETERS 

# URL 
1. HEADER 
2. BODY 
3. METHOD
